# Lab Instructions - Week 8

## Goal

Diagnose a broken async server workflow and explain the fix professionally.

## Workflow

1. Run `.\gradlew.bat test`.
2. Run `.\run-all-demos.ps1`.
3. Read `MIDTERM_PACKET.md`.
4. Run `.\gradlew.bat studentCheck` to see expected TODO failures.
5. Implement one exercise at a time.
6. Complete `DIAGNOSIS_REPORT.md` and `PERFORMANCE_TRADEOFF.md`.
7. Prepare the five-minute explanation using `RECORDED_EXPLANATION_RUBRIC.md`.

## Required Explanation

Be ready to explain:

- what is broken
- what evidence proves it
- what mechanism causes it
- what fix you made
- what the fix costs
- why Kafka-style logs help later distributed systems recover and replay
